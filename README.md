### Reference Documentation

For further reference, please consider the following sections:

JSON format:

```

ts:  timestamp

pt:  processing time

si:  session ID

uu:  user UUID

bg:  business UUID

sha: sha256 of the file

nm:  file name

ph:  path

dp:  disposition (valid values: MALICIOUS (1), CLEAN (2), UNKNOWN (3))

```
